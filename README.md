# Платформа онлайн‑школы (аналог SIRIUS)

Ниже — структурированная концепция платформы онлайн‑обучения, где преподаватели могут зарабатывать, а ученики — получать дополнительные знания дистанционно.

## Цели продукта

- Обучение детей и подростков в дистанционном формате.
- Создание устойчивой экономики для преподавателей (оплата за уроки, курсы, подписки).
- Прозрачная и безопасная среда для родителей/опекунов.

## Ключевые роли

- **Ученик**: проходит курсы, посещает онлайн‑уроки, выполняет задания, получает обратную связь.
- **Преподаватель**: создает курсы, ведет занятия, получает оплату.
- **Родитель/опекун**: контролирует расписание, прогресс и платежи.
- **Администратор**: модерирует контент, управляет качеством и безопасностью.

## Основные возможности (MVP)

### Для учеников

- Регистрация и профиль с указанием возраста, интересов и уровня.
- Каталог курсов и занятий с фильтрами (возраст, предмет, уровень, формат).
- Личный кабинет: расписание, домашние задания, прогресс.
- Встроенная видеосвязь (уроки) и чат с преподавателем.

### Для преподавателей

- Профиль преподавателя с портфолио и рейтингом.
- Конструктор курса: модули, уроки, материалы, задания.
- Управление расписанием и наборами групп.
- Финансовый кабинет: начисления, выплаты, история транзакций.

### Для родителей

- Привязка аккаунта ученика.
- Контроль посещаемости и прогресса.
- Платежный кабинет и отчеты.

### Для администраторов

- Модерация профилей и контента.
- Управление жалобами, блокировками и безопасностью.
- Аналитика по активным пользователям, доходу, удержанию.

## Модель монетизации

- Комиссия платформы с каждого урока или покупки курса.
- Подписка на пакеты занятий (например, 8–12 уроков в месяц).
- Дополнительные сервисы: сертификаты, олимпиадные программы, консультации.

## Безопасность и соответствие

- Родительский контроль и согласие на обработку данных.
- Запись уроков и хранение истории общения (по правилам приватности).
- Модерация чатов и автоматический фильтр контента.

## Рекомендуемая архитектура (MVP)

- **Frontend**: Web‑приложение (React/Vue) + адаптивный дизайн.
- **Backend**: REST/GraphQL API, микросервисы для платежей, контента, расписания.
- **Видео**: интеграция с готовыми решениями (например, WebRTC‑провайдер).
- **Платежи**: интеграция с платежным шлюзом (банковские карты/кошельки).

## Этапы запуска

1. **Discovery**: интервью с родителями и преподавателями, формирование ценности.
2. **MVP**: запуск ключевых функций (каталог, занятия, платежи).
3. **Пилот**: 2–3 направления (например, математика, программирование, физика).
4. **Рост**: расширение курсов, маркетинг, партнерства.

## Что нужно уточнить для старта

- Целевая аудитория: возраст, предметы, языки обучения.
- География и требования законодательства.
- Предпочтительная модель оплаты (разовые занятия, подписка).
- Формат уроков (индивидуальные/групповые, длительность).
